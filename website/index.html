What you get

One global guard

CompositeAuthGuard registered via APP_GUARD handles @Public skip, JWT validation, and permission checks in one pass. No per-handler stacking.

OAuth2-style permissions

<RESOURCE>_READ, _WRITE, _MANAGE for every resource you declare. WRITE covers create + update + delete.

Decorator-driven

@Public, @RequirePermission(...), @CurrentUser() are the entire auth surface in your handlers.

Role matrix

SUPER_ADMIN, ADMIN, USER baked in. Add custom roles at scaffold time. ASSIGNABLE_ROLES guards privilege escalation.

Refresh + throttle

Refresh-token endpoint, ThrottlerGuard, and global ValidationPipe with whitelist + forbidNonWhitelisted.

ORM agnostic

Pick Mongoose, TypeORM, Prisma, or generate a stub repository to wire later. Templates render the right shape.

Living docs

Appends a CLAUDE.md auth section and creates docs/auth/EXTENDING.md — a runbook the next agent uses to extend safely.

Multi-tenant ready

Optional org claim in JWT and organizationId on DTOs. Toggle at scaffold time, no rewrite.

Commands

CommandPurpose
/bootstrap-authFull 5-phase scaffold: analyze → ask → confirm → execute → verify.
/analyze-authPhase 1 only. Reports project state without writing files.
/extend-authAdd a resource, role, or recipe to an existing install via EXTENDING.md.

Auto-triggers on phrases like “add JWT auth”, “scaffold authentication”, “necesito auth en mi proyecto NestJS”, “crea sistema de roles y permisos”.

How it works

  1. Analyze — inspects package.json, ORM, folder layout, existing auth artifacts.
  2. Ask — adaptive clarification: resources, roles, multi-tenancy, target paths.
  3. Confirm — single block: files, deps, final enum shapes. Waits for OK.
  4. Execute — installs deps, renders templates, merges app.module.ts / main.ts / .env.example.
  5. Verify — runs tsc --noEmit, lists endpoints, suggests cURL checks.

No file is written before Phase 4. Each phase blocks until acknowledged.

Non-negotiable design decisions

  • One global guard. No per-handler @UseGuards() stacking.
  • Single login endpoint POST /auth/login for all roles. Distinction by permission, not by route.
  • Permissions follow OAuth2 scope shape: WRITE = create + update + delete; MANAGE = admin ops.
  • Always included: refresh tokens, ThrottlerGuard, global ValidationPipe.
  • Not included by default: account lockout, secondary auth strategies, audit logging — recipes live in the generated EXTENDING.md.
